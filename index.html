API error reference

Every UnitCore CRM API error response follows RFC 7807. The code field is the canonical, stable identifier for the error class and is what you should match in client code. Pages below describe each code, its typical causes, and resolution steps.

Example response shape

HTTP/1.1 409 Conflict
Content-Type: application/problem+json

{
  "type": "https://docs.unitcore.app/errors/resource/duplicate",
  "title": "Duplicate resource",
  "status": 409,
  "code": "resource.duplicate",
  "detail": "A site with slug \"acme\" already exists.",
  "traceId": "00-9b1c1e4d...-01"
}

When matching errors, use code — never title or detail, which are localized for display.
