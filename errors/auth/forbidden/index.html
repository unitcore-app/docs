Forbidden

403

auth.forbidden

You do not have permission to access this resource.

Common causes

  • The user is authenticated but lacks the required permission.
  • The endpoint has `[Authorize]` but no `[RequirePermission]` — strict mode rejects it by default.
  • Calling a Hub-only endpoint from a client tenant.

How to resolve

  • Confirm the user's role grants the required permission.
  • If the endpoint is missing `[RequirePermission]`, raise an audit ticket — strict mode is on in production.
