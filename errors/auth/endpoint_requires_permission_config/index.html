Endpoint missing permission config

500

auth.endpoint_requires_permission_config

An authenticated endpoint is missing its `[RequirePermission]` configuration; strict mode rejects it.

Common causes

  • An endpoint has `[Authorize]` but no `[RequirePermission]` — strict mode (production default) refuses to serve it.
  • Developer added the route but forgot to declare the permission.

How to resolve

  • Add the appropriate `[RequirePermission("...")]` attribute to the endpoint.
  • File an audit ticket — this is the strict-permissions endpoint gap class of bug.
