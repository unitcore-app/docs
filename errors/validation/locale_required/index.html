Locale required

400

validation.locale_required

The endpoint requires an explicit locale and none was provided.

Common causes

  • Missing `Accept-Language` header or locale path segment.
  • Token has no default locale claim.

How to resolve

  • Send an `Accept-Language` header with a supported BCP 47 locale (e.g. `en`, `uk`).
  • Set a default locale on the user profile.
