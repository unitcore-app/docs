Field has an invalid value

400

validation.field_invalid

A field was present but its value did not pass type or format checks.

Common causes

  • Wrong data type (string where number expected, etc.).
  • Value outside an allowed enum / range.
  • Malformed GUID, date, or email.

How to resolve

  • Inspect the `errors` array — each entry includes the field path and the failed rule.
  • Normalize the value client-side before sending.
