Concurrency conflict

409

resource.concurrency_conflict

The resource was modified by another request. Please reload and try again.

Common causes

  • Optimistic concurrency token (rowversion / updatedAt) did not match server state.
  • Two clients edited the same record at the same time.

How to resolve

  • Re-fetch the resource to get the latest concurrency token.
  • Re-apply your changes on top of the fresh copy and submit again.
