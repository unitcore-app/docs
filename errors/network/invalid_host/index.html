Invalid host

400

network.invalid_host

The `Host` header is not on the API's allow-list.

Common causes

  • Request is hitting the API through an unexpected DNS name.
  • Host header was rewritten or spoofed by an intermediary.

How to resolve

  • Use one of the configured hostnames (see `Security__AllowedHosts`).
  • Operator: add the missing host to `values.security.extraAllowedHosts`.
